Fluxo de Dados — Bater Ponto

// Colaborador clica no botão de ponto doPunch() // punch.js → getCoords() // tenta geolocalização (5s timeout) → dbAddPunch() // INSERT punch_records (lat/lng só se não null) → state.PS[idx].push(rec) → renderPunch() // atualiza UI // Ao finalizar expediente doExit() → getCoords() → dbAddPunch() → calcula saldo do dia → dbUpdateEmployeeStats() // UPDATE employees SET bank_minutes, worked_hours... // Gestor abre Editar Registros renderEdit() → renderPunchMap(recs) → cria mapa Leaflet com pins entrada (verde) / saída (vermelho) → geocode(lat, lng) // Nominatim → "Rua Tal, 123 — Cidade"

Segurança

  • ✓RLS ativo — colaborador acessa apenas os próprios dados no Supabase
  • ✓Colaborador não pode deletar registros de ponto (UI + JS + RLS)
  • ✓Edge Function create-user — só gestor autenticado cria novos usuários Auth
  • ✓Anon key no frontend (nunca a service_role key)
  • ✓Variáveis de ambiente via Vercel env — nunca no código-fonte

Avisos Importantes

⚠️ A service_role key do Supabase nunca deve ir para o frontend. Só usar em scripts/seed.js e Edge Functions no servidor.
📋 Ao executar o SQL de reset (DELETE FROM punch_records), os registros são permanentemente apagados. Não há desfazer.
